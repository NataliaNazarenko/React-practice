React Native

React Native — фреймворк інтерфейсу користувача з відкритим кодом, створений Meta Platforms, Inc Він використовується для розробки програм для Android Android TV,, iOS,, macOS, , tvOS, ,Web, ,Windows ,і UWP дозволяючи розробникам використовувати фреймворк React разом із нативними можливостями платформи. Він також використовується для розробки програм віртуальної реальності в Oculus.

В останні роки фреймворк набув величезної популярності тому що:

- Прискорює швидкість розробки
- Скорочує час виходу продукту на ринок
- Дає можливості покривати декілька платформ без додаткових вкладень
- Дає можливість використовувати спільні рішення для мобільних та вебдодатків
- Наявність гігантської та активної спільноти
- Переваги відкритого коду та репозиторію NPM (понад 1 млн модулів)
- Менша команда порівняно з Native-розробкою
- Використовує JavaScript, що є найпопулярнішою мовою програмування у світі

Додатки на React Native — безпечні, швидкі і легко підтримувані, майже як нативні (орієнтовані на роботу з конкретною системою, такі що володіють доступом до апаратної частини, наприклад, камері або мікрофона), а крім того відкривають нові можливості кастомізації дизайну. Проєкти на RN можуть виглядати так, як вам хочеться до найменших деталей, що неможливо у випадку з написаними на офіційних мовах програмування для операційних систем Apple і Google.

Єдиний випадок, коли вам точно не підійде React Native — якщо ви не збираєтеся запускати проєкт відразу ж або не готові платити за подальшу підтримку. Фреймворку трохи більше ніж 5 років, що дуже мало — деяких щодо базових компонентів в ньому немає і поновлення, за якими потрібно стежити, з'являються дуже часто. Це добре в перспективі і говорить про популярність рішення, але зараз не можна зробити проєкт і запустити його через пів року.

До цього моменту доведеться оновлювати сам фреймворк і використані компоненти бібліотек, що зводить нанівець всю економію часу і грошей, хоча і робиться досить просто, завдяки спеціальним сервісам і React Native tutorial, котрий підказує, що потребує доопрацювання. JavaScript частина проєкту завжди оновлюється автоматично, що прибирає проблему сумісності версій, а ось нативні компоненти в будь-якому випадку потребують фахівця, який буде стежити за їх актуальністю.

Одна з головних причин звернути увагу на React Native — приклади додатків. Писати на цьому фреймворку ігри та програми для роботи з big data не вийде (для них краще вибрати нативні мови), але можна запускати по-справжньому масштабні проєкти.

Середовище розробки

Налаштування середовища розробки залежить від вашої операційної системи нам потрібно зайти на https://reactnative.dev/docs/environment-setup та обрати потрібну операційну систему та платформу на яку ми налаштовуємо.

Рекомендуємо встановити Node і Watchman за допомогою Homebrew.
Виконайте такі команди в терміналі після встановлення Homebrew:
brew install node
brew install watchman

Якщо ви вже встановили Node у своїй системі, переконайтеся, що це Node 14 або новіша версія. Watchman — це інструмент Facebook для спостереження за змінами у файловій системі. Настійно рекомендується встановити його для кращої продуктивності.

Рекомендуємо встановити дистрибутив OpenJDK під назвою Azul Zulu за допомогою Homebrew. Виконайте такі команди в терміналі після встановлення Homebrew:

brew tap homebrew/cask-versions
brew install --cask zulu11

# Get path to where cask was installed to double-click installer brew info --cask zulu11

Після встановлення JDK оновіть змінну середовища JAVA_HOME. Якщо ви виконали наведені вище дії, JDK, імовірно, буде за адресою /Library/Java/JavaVirtualMachines/zulu-11.jdk/Contents/Home

Дистрибутив Zulu OpenJDK пропонує JDK для Intel і M1 Mac. Це гарантує, що ваші збірки будуть швидшими на M1 Mac порівняно з використанням JDK на основі Intel.

Якщо ви вже встановили JDK у своїй системі, ми рекомендуємо JDK 11. Ви можете зіткнутися з проблемами під час використання вищих версій JDK.

Середовище розробки Android

Налаштування середовища розробки може бути дещо втомливим, якщо ви новачок у розробці Android. Якщо ви вже знайомі з розробкою Android, можливо, вам знадобиться налаштувати кілька речей. У будь-якому випадку уважно виконайте кілька наступних кроків.

1. Встановіть Android Studio Завантажте та встановіть Android Studio. Перебуваючи в майстрі інсталяції Android Studio, переконайтеся, що біля всіх наступних пунктів позначено прапорці:

- Android SDK - Android SDK Platform - Android Virtual Device

Потім натисніть «Далі», щоб інсталювати всі ці компоненти.

Встановіть Android SDK

Android Studio за умовчанням встановлює останню версію Android SDK. Однак для створення програми React Native із нативним кодом потрібен, зокрема, Android 13 (Tiramisu) SDK. Додаткові Android SDK можна встановити через SDK Manager в Android Studio

Для цього відкрийте Android Studio, натисніть кнопку «Додаткові дії» та виберіть «Менеджер SDK»

Менеджер SDK також можна знайти в діалоговому вікні «Параметри» Android Studio в розділі «Вигляд і поведінка» → «Параметри системи» → «Android SDK».

Виберіть вкладку «Платформи SDK» у диспетчері SDK, а потім поставте прапорець «Показати деталі пакета» у нижньому правому куті. Знайдіть і розгорніть запис Android 13 (Tiramisu), а потім переконайтеся, що вибрано такі пункти:

Платформа Android SDK 33
Образ системи Intel x86 Atom_64 або образ системи Google API Intel x86 Atom або (для Apple M1 Silicon) образ системи Google API ARM 64 v8a
Далі виберіть вкладку «Інструменти SDK» і поставте тут прапорець поруч із «Показати деталі пакета». Знайдіть і розгорніть запис «Android SDK Build-Tools», а потім переконайтеся, що вибрано 33.0.0

Нарешті натисніть «Застосувати», щоб завантажити та інсталювати Android SDK і відповідні інструменти збірки.

Налаштуйте змінну середовища ANDROID_HOME Інструменти React Native вимагають налаштування деяких змінних середовища, щоб створювати програми з нативним кодом.

Додайте такі рядки до файлу конфігурації ~/.zprofile або ~/.zshrc (якщо ви використовуєте bash, то ~/.bash_profile або ~/.bashrc):

export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/platform-tools

Запустіть джерело ~/.zprofile (або джерело ~/.bash_profile для bash), щоб завантажити конфігурацію у поточну оболонку. Переконайтеся, що ANDROID_HOME встановлено, запустивши echo $ANDROID_HOME, і відповідні каталоги додано до вашого шляху, запустивши echo $PATH.

Переконайтеся, що ви використовуєте правильний шлях Android SDK. Ви можете знайти фактичне розташування SDK у діалоговому вікні «Параметри» Android Studio в розділі «Вигляд і поведінка» → «Параметри системи» → «Android SDK».

Власний інтерфейс командного рядка React
React Native має вбудований інтерфейс командного рядка. Замість того, щоб установлювати та керувати певною версією CLI глобально, ми рекомендуємо отримати доступ до поточної версії під час виконання за допомогою npx, який постачається з Node.js. З npx react-native поточна стабільна версія CLI буде завантажена та виконана під час виконання команди.

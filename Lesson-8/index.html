Вступ

Один із важливих аспектів розробки веб-додатків на основі React - це ефективна та гнучка стилізація компонентів. У традиційному підході до стилізації, CSS правила зазвичай розташовуються у зовнішніх файлах і підключаються до HTML-сторінки. Проте, в React є кілька підходів, які дозволяють зручно та ефективно використовувати стилі для компонентів. Один з найпоширеніших підходів до стилізації в React - це використання CSS-класів та звичайного CSS для оформлення компонентів. За допомогою класів можна додавати стилі до компонентів та використовувати CSS-селектори для точного визначення стилів. Цей підхід дозволяє легко керувати стилями та повторно використовувати їх у різних компонентах. Однак, разом з ростом складності додатків React, з'являються нові вимоги до стилізації. Тут на сцену виходять інші підходи, такі як CSS модулі та CSS-in-JS. Використання CSS модулів дозволяє локалізувати стилі для кожного компонента, уникнути конфліктів між ними та забезпечити більшу переносимість коду. З іншого боку, CSS-in-JS підхід дає змогу вбудовувати стилі безпосередньо в компоненти, використовуючи JavaScript. Це дозволяє створювати динамічні та перефекціоновані компоненти, які можуть реагувати на зміни стану додатку.

Вбудовані стилі

Вбудовані стилі - це один із способів задавання стилів для веб-сторінок. Замість того, щоб розміщувати CSS-код у зовнішньому файлі або вбудовувати його у HTML-документ за допомогою тега style, при використанні інлайнових стилів CSS правила задаються безпосередньо в атрибутах HTML-елементів. Це дозволяє зберігати стилі разом з контентом, що полегшує їх редагування та підтримку.

Вбудовані стилі в React - це можливість встановлювати стилі безпосередньо в атрибуті "style" HTML-елементу. Замість використання зовнішніх файлів CSS або CSS-класів, ви можете встановлювати стилі безпосередньо в JSX коді.

Для використання inline стилів в React, вам потрібно передати об'єкт стилів як значення атрибуту "style". У цьому об'єкті ви вказуєте назву властивості CSS як ключ та відповідне значення стилю.

Можна виділити кілька обов'язкових правил вбудованих стилів:

- Імена властивостей, що складаються з двох і більше слів, наприклад background-color, обов'язково повинні бути записані в camelCase нотації (backgroundColor), як при зверненні до властивостей об'єкта style у DOM-елемента.
- До числових значень більшості властивостей буде автоматично додано суфікс px. Якщо необхідно використовувати одиниці відмінні від px, або значення складається з кількох частин, воно вказується як рядок.

Для використання рендеру за умовою в inline стилях вам потрібно динамічно формувати об'єкт стилів залежно від умови. Можна зробити це шляхом використання тернарного оператора або умовного виразу для визначення стилів.

Вбудовані стилі можуть видаватися зручними через простоту використання, але вони мають ряд істотних недоліків.

• Дуже слабка масштабованість та повторне використання стилів
• Обмежені можливості (псевдокласи, псевдоелементи, адаптивні властивості)
• Погана продуктивність при відмальовуванні досить великої кількості елементів
• Відсутність зручних інструментів розробника

На практиці вбудовані стилі використовуються тільки для динамічно обчислюваних значень властивостей, у парі із зовнішніми таблицями стилів. В якості основного способу стилізації компонентів вбудовані стилі не використовуються в проектах та тому не рекомендуються.

Звичайний css

CSS (Cascading Style Sheets) є мовою стилів, яка використовується для оформлення веб-сторінок. Зазвичай CSS-код розміщується у зовнішніх файлів, які підключаються до HTML-документів. Завдяки CSS можна задавати різноманітні стилі для елементів сторінки, такі як кольори, шрифти, розташування, розміри тощо. CSS також дозволяє створювати анімації та визначати реакцію елементів на події користувача.

Ви можете підключити зовнішні файли CSS безпосередньо в компоненті React, використовуючи тег link або import.

Бібліотеки clsx, classNames

Для вирішення більшості завдань, пов'язаних з безліччю класів, що задаються згідно з певними умовами, використовують бібліотеки “clsx” чи “classNames”. Звичайно, JavaScript надає багатий синтаксис, але здебільшого пишуться непродуктивні рішення або код, що погано читається. Ці бібліотеки стандартизують цей процес і роблять його більш зручним за рахунок продуманого синтаксису.

css - модулі

CSS-модулі - це не офіційна специфікація, вони не імплементовані в браузери. Це процес, що запускається на стадії складання проекту (наприклад, за допомогою Webpack), в результаті виконання якого імена класів замінюються на унікальні. Це дозволяє використовувати одне і те ж ім'я класу в різних CSS-файлах, не турбуючись про конфлікти імен. Цей підхід був розроблений, щоб вирішити проблему глобальної області видимості в CSS.
Синтаксис імпорту CSS-модуля нагадує імпорт файлу JavaScript модуля. У CSS-модулі є експорт за замовчуванням - об'єкт відповідності оригінального та згенерованих імен класів. У фінальному файлі стилів буде унікальне ім'я класу у форматі [filename]_[classname]__[hash].

Створення CSS-модуля:

Створіть файл стилів з розширенням .module.css або .module.scss. Наприклад, MyComponent.module.css. В цьому файлі ви можете визначати стилі для вашого компонента.

Використання CSS-модуля в компоненті:

Імпортуйте стилі з CSS-модуля і використовуйте їх як класи в JSX елементах вашого компонента.

CSS-модулі дозволяють створювати локальні, ізольовані стилі для кожного компонента, зменшуючи ймовірність конфліктів між класами та забезпечуючи більшу переносимість коду. Крім того, вони спрощують підтримку і рефакторинг стилів, оскільки вони знаходяться поруч з компонентами, які вони стилізують.

Бібліотека classNames також може бути використана разом з CSS-модулями для зручного управління класами в компонентах React. Вона дозволяє комбінувати та умовно додавати класи з CSS-модулів.
Імпортуйте бібліотеку classNames та стилі з CSS-модуля. Використовуйте їх для генерації класів для елементів JSX.

CSS-in-JS

CSS-in-JS - це підхід до стилізації веб-додатків, де стилі описуються та застосовуються безпосередньо в JavaScript-коді, використовуючи об'єктні або функціональні підходи. Замість того, щоб мати окремі файли CSS, стилі вбудовуються безпосередньо в компоненти.

Styled Components: Це одна з найпопулярніших бібліотек для CSS-in-JS. Вона дозволяє визначати стилі в рядку шаблонних літералів та застосовувати їх до компонентів.

Emotion: Це потужна бібліотека CSS-in-JS, яка дозволяє використовувати стилі як об'єкти або функції. Вона підтримує динамічні стилі, генерацію унікальних класів та інші функціональні можливості.

JSS (JavaScript Style Sheets) - це бібліотека, яка дозволяє використовувати JavaScript для опису стилів. Замість того, щоб використовувати CSS файли або рядки шаблонів, ви можете описувати стилі прямо в коді JavaScript.

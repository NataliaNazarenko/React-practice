Створення екранів.

Імпортування необхідних компонентів:
Імпортуємо React та Component з бібліотеки react.
Імпортуємо View, Text, StyleSheet з react-native

Створення компонента для екрану:
Створюємо новий клас, який розширює Component.
У класі визначаємо метод render(), який буде повертати вміст екрану.

Визначення розмітки екрану:
У методі render() використовуємо компонент View для створення контейнера, в якому розташовуватимуться інші елементи.
Додаємо дочірні елементи, такі як Text, для відображення тексту на екрані. Використовуємо StyleSheet.create() для визначення стилів компонентів.

Експортування компонента: В кінці файлу експортуємо створений компонент за допомогою export default

Використання екрану в додатку:
У файлі, який відповідає за головний компонент додатку (наприклад, App.js), імпортуємо створений компонент екрану. Використовуємо створений компонент в розмітці додатку, наприклад, за допомогою ScreenComponent

Це загальний опис процесу створення екранів у додатку на React Native. Зверни увагу, що це лише загальні кроки, і більш детальна реалізація залежить від потреб та вимог конкретного додатку.

Бібліотеки

У React Native існує кілька популярних бібліотек, які допомагають створювати екрани та навігацію в додатках. Ось декілька з них:

React Navigation: Це найпопулярніша бібліотека для навігації в React Native. Вона надає широкі можливості для створення різних типів навігаційних елементів, таких як стекова навігація, таб-бари, випадаючі меню та багато іншого

React Native Navigation: Ця бібліотека є альтернативою React Navigation. Вона пропонує низькорівневу навігацію, що базується на нативних компонентах платформи. Це робить її дещо швидшою із великою кількістю елементів навігації.

React Native Router Flux: Це проста бібліотека, яка забезпечує легку інтеграцію навігації в додатки React Native. Вона використовує підхід заснований на роутингу (шляхах), де кожен екран відповідає окремому роуту.

React Native Elements: Ця бібліотека надає готові компоненти із стилізацією, такі як кнопки, картки, списки, іконки тощо. Вона спрощує процес створення інтерфейсу та вирішує багато рутинних задач.

NativeBase: Це ще одна бібліотека компонентів, яка пропонує готові елементи UI, включаючи заголовки, кнопки, форми, списки тощо. Вона також надає підтримку тем і стилів.

Це лише кілька прикладів бібліотек для створення екранів у додатках на React Native. Залежно від потреб проекту, ви можете вибрати ту, яка найбільше відповідає вашим вимогам та зручна для використання.

Ми будемо використовувати React Navigation:
https://reactnavigation.org/docs/getting-started/

Встановлення бібліотеки React Navigation

Відкрийте командний рядок і перейдіть до кореневої папки вашого проекту. Виконайте команду npm install @react-navigation/native для встановлення основних пакетів. Встановіть додаткові пакети.

Налаштування залежностей:
Для платформи Android виконайте команду npx react-native link для підключення потрібних пакетів. Для платформи iOS виконайте команду cd ios && pod install && cd .. для оновлення залежностей.

Створення основного навігаційного контейнера:
Створіть новий файл, наприклад AppNavigator.js, і імпортуйте необхідні компоненти: import { NavigationContainer } from '@react-navigation/native';.

Визначте компонент, який буде відповідати за основний навігаційний контейнер

Створення стекової навігації: Внутрішній контент NavigationContainer буде містити навігаційні екрани. Створіть новий файл, наприклад MainStack.js, і імпортуйте необхідні компоненти: import { createStackNavigator } from '@react-navigation/stack';. Визначте стекову навігацію та її екрани.

Створення екранів:
Створіть окремі файли для кожного екрану, наприклад HomeScreen.js і DetailsScreen.js. В цих файлах визначте компоненти екранів, які будуть відображатися. Не забудьте імпортувати необхідні компоненти з react-native та reactnavigation.

Використання навігації в головному компоненті додатку:
У файлі, який відповідає за головний компонент додатку (наприклад, App.js), імпортуйте AppNavigator і використовуйте його в розмітці додатку.

Для переключення екранів за допомогою бібліотеки React Navigation на React Native використовується функція навігації navigate або компонент Touchable для навігаційних елементів.

Ось кілька способів переключення екранів:
Використання функції навігації navigate:
В компоненті, який має кнопку або подібний елемент, імпортуйте useNavigation з @react-navigation/native. Визначте змінну navigation за допомогою useNavigation(). Використайте navigation.navigate('ScreenName') для переключення на інший екран, де 'ScreenName' є ім'ям екрану, яке ви хочете відкрити.

Використання компонента Touchable:
Використовуйте один із компонентів TouchableOpacity, TouchableHighlight або TouchableWithoutFeedback з react-native. Визначте обробник події, наприклад, onPress, та викличте функцію навігації navigate у відповідності до вашої логіки.

Для передачі даних між екранами за допомогою бібліотеки React Navigation на React Native можна використовувати параметри навігації або створювати спеціальний контекст для передачі даних. Ось кілька способів передачі даних:
Використання параметрів навігації:
При переключенні на новий екран за допомогою navigation.navigate('ScreenName' , { param: value }), можна передавати параметри у вигляді об'єкта. На екрані приймаючого компонента можна отримати передані параметри з route.params.

Використання контексту:

Створіть новий файл, наприклад DataContext.js, в якому створіть контекст даних за допомогою createContext(). Визначте провайдер контексту, який буде зберігати дані. Використовуйте провайдер контексту в основному навігаційному компоненті та передавайте дані через значення контексту. На екранах, які мають доступ до контексту, використовуйте useContext() для отримання переданих даних. Наприклад: Це два основні способи передачі даних між екранами за допомогою бібліотеки React Navigation на React Native. Ви можете використовувати ці підходи залежно від ваших потреб та вимог.
